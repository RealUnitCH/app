RealUnit Wallet — Handbook

User-Doku und Test-Doku in einem. Jeder Screenshot auf dieser Seite ist die Ausgabe eines Maestro-Tier-3-Flows (.maestro/<spec>.yaml), der auf einem iOS-Simulator gegen das Debug-Bundle der App läuft. Wenn das Handbook aktuell aussieht, ist die App es auch — und umgekehrt.

📱 App: RealUnit Wallet (Flutter) 📂 Code: DFXswiss/realunit-app 🧪 Tests: ~1100 (Tier 0/1) · 1 Tier-3 flow (Welcome)

Single Source of Truth

Die Flows liegen in .maestro/*.yaml und sind mit tags: [handbook] markiert. Reproduzieren:

flutter build ios --simulator --debug
xcrun simctl install booted build/ios/iphonesimulator/Runner.app
scripts/run-handbook-flows.sh

Das Skript führt alle handbook-Flows aus und legt die Screenshots in docs/handbook/screenshots/ ab. Die HTML-Seite linkt jedes Bild aus diesem Verzeichnis — neu generieren = Handbook ist aktuell.

Die fünf Test-Tiers sind in docs/testing.md dokumentiert. Tier 3 (Maestro auf echter Hardware) ist offiziell deferred — dieses Handbook nutzt den Simulator-Weg, der heute schon funktioniert.

Heute: nur ein einziger Flow (01-Welcome) als Proof. Weitere Specs werden inkrementell ergänzt. Jeder weitere Flow ist eine zusätzliche .maestro/NN-name.yaml + ein neuer <section class="spec">-Block in diesem File.

01Welcome & Onboarding

.maestro/01-welcome.yaml
3 Screens
Flow ↗

Der allererste Screen, den ein neuer User sieht. Zwei Wege gehen davon ab — die App-Wallet (Seed-Phrase auf dem Gerät) und die BitBox-Wallet (Hardware Wallet aus der Schweiz). Auf dem App-Weg gibt es wiederum drei Optionen: neue Wallet anlegen, vorhandene wiederherstellen, oder im Debug-Modus eine bestimmte Adresse via Signatur freischalten.

01-welcome L7
Der Splash-Screen mit RealUnit-Logo, dem Claim „Bankenunabhängig. Sicher.", einem Bergpanorama im Hintergrund und dem Start-CTA. Unter dem Button steht der i18n-Disclaimer, dass die Nutzung der App die Annahme der Nutzungsbedingungen impliziert.
02-create-vs-restore L11
Nach Start wählt der User den Wallet-Typ: Digitale Wallet (App) für eine reine Software-Wallet auf dem Gerät, oder BitBox für die Hardware-Wallet aus der Schweiz. Beide Kacheln tragen Subtitle-Text + Icon und tappen weiter in den jeweiligen Onboarding-Pfad.
03-software-wallet-terms L15
Innerhalb des App-Wallet-Zweigs gibt es drei Optionen: Neue Wallet erstellen (frische BIP-39-Mnemonic), Wallet wiederherstellen (Import einer bestehenden Phrase, z.B. von Aktionariat), und Adresse + Signatur als Debug-Pfad für die Backend-Integration.
